# 读 Satoshi 比特币白皮书：比特币没有消灭信任，而是重新设计了信任

读完 Satoshi Nakamoto 的《Bitcoin: A Peer-to-Peer Electronic Cash System》后，我最深的感受并不是“比特币是一种去中心化货币”，而是：比特币并没有真正消灭信任，而是把原来对银行、支付机构和中心服务器的信任，转化成了对公开规则、密码学证明、计算成本和多数参与者行为的信任。在传统电子支付系统中，我们之所以敢相信一笔转账已经完成，是因为银行维护着一份中心化账本。银行可以判断付款人的余额是否足够，也可以阻止同一笔钱被重复支付。如果发生争议，银行还可以查询后台记录并作出最终裁决。因此，传统支付系统实际上依赖一个被普遍认可的中心机构。Satoshi 想解决的问题是：如果没有银行这样的中心机构，互联网上两个互不认识的人，能不能直接完成一笔可靠的电子支付？这个问题比我原先想象的复杂。最初我认为，只要使用数字签名，就可以保证电子货币的安全。付款人用自己的私钥签名，其他人用公钥验证，就能证明交易确实由付款人发出。但是读完论文后，我意识到，数字签名只能证明“谁同意了这笔交易”，却不能证明“这笔钱是否已经被花过”。例如，Alice 拥有一笔数字货币，她完全可以签署两笔交易，一笔转给 Bob，另一笔转给 Charlie。这两笔交易的签名都可能是真的。问题不在于签名是否被伪造，而在于系统必须决定哪一笔交易先发生，哪一笔应该被接受。在银行系统中，这个问题很简单，因为银行只维护一份账本。第一笔交易执行后，余额已经被扣除，第二笔交易自然会被拒绝。但在点对点网络中，不同节点收到交易的时间可能不同。有些节点先看到 Alice 向 Bob 的付款，另一些节点可能先看到 Alice 向 Charlie 的付款。如果没有统一的历史顺序，整个系统就无法判断哪笔交易有效。因此，我认为，比特币真正困难的地方不是“如何制造一种不能复制的数字文件”，而是“如何让全网对交易发生的顺序形成共同认识”。数字信息本来就可以被复制，比特币也没有试图阻止交易数据被复制。它解决的是：即使同一笔钱出现了两次支出，网络仍然能够逐渐选择其中一笔，并把另一笔排除在共同账本之外。为了解决这个问题，比特币引入了区块、哈希链和工作量证明。节点将一段时间内的交易打包成区块，每个区块都包含前一个区块的哈希值。这样，区块之间就形成了一条连续的链。如果有人试图修改过去某个区块中的交易，该区块的哈希值就会发生变化，后面所有区块与它之间的连接也会失效。不过，仅仅使用哈希链还不够。攻击者修改旧区块后，仍然可以重新计算后续区块。因此，比特币进一步要求每个区块都必须完成工作量证明。矿工需要不断改变 nonce，并反复计算区块哈希，直到找到满足难度条件的结果。这个计算过程没有产生实际的数学结论，但它有一个非常重要的作用：让写入账本和修改历史需要付出真实成本。找到一个有效区块需要进行大量计算，而验证这个区块却非常容易。其他节点只需要计算一次哈希，就可以确认结果是否满足要求。这形成了一种不对称关系：生成证明很昂贵，检查证明很便宜。任何人都可以参与记账，但不能零成本地制造大量虚假历史。我以前把挖矿理解为矿工通过“解题”获得奖励。现在我更愿意把它理解为：矿工通过消耗计算资源，竞争公共账本的临时记账权。工作量证明不是为了证明某个人值得信任，而是为了证明这个人确实为自己提出的账本版本付出了成本。当网络中同时出现两个有效区块时，比特币也不会立刻由某个管理者决定哪个区块正确。不同节点可以暂时在不同分支上继续工作。随着新的区块不断产生，其中一条链会积累更多工作量。节点按照协议选择累计工作量最大的有效链，并继续在这条链上构建新区块。这就是论文中的“最长链”规则。不过我认为，把它理解成“区块数量最多的链”并不完全准确。它真正表达的是：全网选择累计工作量最多的有效历史。这一规则让我对比特币中的“真相”有了新的理解。在传统系统中，真相来自中心机构的最终确认；在比特币中，真相更像是当前最难推翻的历史版本。它不一定具有绝对、瞬间的最终性，而是随着新区块不断增加，变得越来越难以修改。一笔交易刚刚进入区块时，仍然可能因为分叉而被替换。但是，随着后续确认数量增加，攻击者要修改这笔交易，就必须重新完成该区块以及后续所有区块的工作量，同时还要追赶诚实节点继续增长的链。如果攻击者掌握的计算能力低于诚实网络，那么追赶成功的概率会随着确认数量增加而不断降低。因此，比特币提供的不是绝对意义上的不可撤销，而是一种概率性的安全。确认越多，交易被推翻的可能性越低。传统银行可能通过制度和法律宣布一笔交易最终完成，而比特币则通过不断叠加计算成本，使旧交易逐渐获得稳定性。这让我觉得，所谓“区块链不可篡改”其实是一种简化说法。区块链并不是在物理上不能被修改，而是修改之后必须重新完成大量工作。如果攻击者拥有足够强的算力，理论上仍然可能重组部分链。比特币真正做到的不是让攻击变成绝对不可能，而是让攻击变得越来越昂贵，并在正常假设下缺乏经济合理性。除了工作量证明，我认为论文中的激励机制也非常关键。如果系统只要求节点消耗计算资源，却不给任何回报，就很难保证长期有足够多人维护网络。比特币通过新区块奖励和交易手续费，让维护账本成为一种可以获得收益的行为。更重要的是，矿工只有按照规则生成区块，其他节点才会接受这个区块。即使矿工花费了大量算力，如果生成的是无效区块，其他节点仍然会拒绝它，矿工也无法获得有效奖励。因此，协议试图让矿工的个人利益与系统安全保持一致。这一设计并没有假设矿工是无私的。相反，它默认参与者会追求自身利益，然后通过经济奖励，让遵守规则通常比攻击系统更加有利。我认为这是比特币非常现实的一面：它不是依靠所有参与者的道德，而是依靠规则和利益关系约束行为。当然，这套机制并不完美。它依赖于多数有效算力遵守协议，也依赖比特币奖励具有经济价值。如果算力高度集中，少数矿池就可能拥有较大的影响力。如果攻击者的目标不是赚钱，而是愿意花费巨大成本破坏系统，单纯的经济激励也不一定能够完全阻止攻击。所以，比特币的安全不是纯粹的密码学安全，而是由密码学、计算资源、网络传播、经济激励和参与者分布共同构成的。只研究哈希算法或数字签名，并不能完整解释比特币为什么能够运行。读完白皮书后，我也不再认同比特币是一个“完全不需要信任”的系统。使用者仍然需要相信密码学算法在现实中足够安全，相信自己运行的软件正确实现了协议，相信多数算力不会联合攻击网络，也相信点对点网络可以正常传播交易和区块。真正发生变化的是信任的对象。传统支付系统要求用户相信某个具体机构不会随意修改记录，不会滥用权力，也不会发生严重的内部故障。比特币则把这种集中式信任拆分开，转化为对公开代码、公开数据、数学规则和经济成本的信任。任何节点都可以独立验证交易，不需要因为某个机构声称“账本是正确的”就直接接受。这种设计也有明显代价。为了减少对中心机构的依赖，比特币需要让大量节点重复验证交易，需要通过工作量证明消耗资源，还需要等待多个区块才能获得较高的交易确定性。相比中心数据库，比特币并不一定更快，也不一定更节省资源。因此，我认为，比特币不是简单地证明“去中心化一定比中心化更好”，而是展示了一种不同的工程选择：如果人们不愿意把最终记账权交给单一机构，就必须接受更高的计算成本、更低的处理效率以及概率性的最终确认。这也是我读完论文后最真实的理解。比特币最重要的创新，不是创造了一种价格不断变化的数字资产，而是提出了一种新的公共记账方式。数字签名负责证明交易授权，点对点网络负责传播信息，哈希链负责连接历史，工作量证明负责提高修改成本，累计工作量规则负责解决分叉，经济激励则吸引参与者维护系统。这些技术单独来看并不是全新的，但 Satoshi 把它们组合成了一套可以在陌生人之间运行的机制。在这套机制中，没有任何单个节点被默认视为绝对可信，但每一笔交易和每一个区块都可以被所有人验证。所以，我最终认为：比特币并没有让信任消失，而是把“相信一个中心机构”，改造成了“相信一套任何人都可以检查的规则，并相信伪造历史需要付出足够高的成本”。相比比特币的市场价格，这种关于信任、成本和公共历史的设计，才是这篇白皮书真正值得阅读的地方。
